美术资产在 `assets/`（概念图=美术圣经、角色三视图、四张已验缝的无缝贴图）。git 历史保留了全部决策过程。

## 产品红线（这些是刻意设计，不是 bug，不许"修复"）

1. **无账号、无登录、无邮箱**。归属只靠 Ed25519 密钥对：私钥（墓钥）浏览器本地生成、永不上传、只显示一次；服务器只存公钥。丢钥即永久失去碑的控制权——这是产品承诺，不做找回功能。
2. **石子永远不显示数字**。只做视觉堆积。不做点赞、评论、热榜、排行、关注、任何社交计数。
3. **路过是加权随机，权重偏向访问少的碑**（被走到的概率与热度反相关）。不做热门推荐、不做按热度排序的任何入口。
4. **全站不出现技术错误码**。所有面向用户的失败文案用守墓人的口吻（中文、克制、庄重）。文案改动需产品负责人（佳欢）确认。
5. **守墓人审核不可绕过**：下葬必经入园审；审核 API 失败时碑进 buried 停灵态等夜巡补审，宁停不漏。危机类内容温和拒葬并附求助资源，不冷拒。
6. **美学宪法**：颜色、光照、气质以 `assets/concept/` 两张概念图为准；风格化低多边形，禁止照片级素材、禁止恐怖元素。昼夜跟访客本地真实时间。
7. **David 的记忆有隐私门槛**：只有访客凭墓钥在会话内认领过碑，对话 API 才注入其碑文；否则只聊园子。

## 边界与分工

- 本仓库与「念念」是两个独立产品，不共享代码库、数据库与用户体系。David 的对话机制与念念理念同源，但实现各自独立。
- Feishu 告警管线：产品负责人已有现成 webhook（AI观察项目），问他要地址即可，不要新建。
- 密钥/环境变量清单见开发文档第五节，全部由产品负责人提供，不要硬编码。
- 所有密钥只进环境变量；Anthropic key 只在服务端 API routes 使用；夜巡端点必须带 secret 参数校验。

## 完成定义

每个里程碑的验收标准写在开发文档里。提交粒度：一个里程碑内小步提交，信息用中文，说清楚"改了什么、为什么"。拿不准的决策（尤其涉及文案、美学、红线边缘）停下来问，不要替产品做决定。
